This documentation describes the extensions and additions that have been made to the Java Debug Interface (JDI) in order to allow debugging of F3 programs. These extensions and additions are called F3-JDI.

F3-JDI is implemented as a set of classes that wrap the normal JDI implementation classes. These F3-JDI classes implement the JDI interfaces so that debuggers will in general just access the methods defined in the JDI interfaces. Debuggers will only need to access these F3-JDI implementation classes directly in order to use the new classes and methods that have been added and thus are not in the JDI interfaces.

F3-JDI is experimental and subject to change in future releases.

The F3-JDI classes are included in the file f3jdi.jar. In order for a client to use F3-JDI, f3jdi.jar must be on the client's classpath. The client should first get a {@link com.sun.jdi.VirtualMachineManager} object:

com.sun.jdi.VirtualMachineManager vmm = org.f3.jdi.F3Bootstrap.virtualMachineManager();

This gives the client a F3 enabled VirtualMachineManager (org.f3.jdi.F3VirtualMachineManager). The client can then get one of the following F3-JDI connectors from the VirtualMachineManager:

  • org.f3.jdi.connect.F3LaunchingConnector (this is the default connector)
  • org.f3.jdi.connect.F3ProcessAttachingConnector
  • org.f3.jdi.connect.F3RawLaunchingConnector
  • org.f3.jdi.connect.F3SocketAttachingConnector
  • org.f3.jdi.connect.F3SocketListeningConnector
  • org.f3.jdi.connect.F3SharedMemoryAttachingConnector - currently only available on Windows
  • org.f3.jdi.connect.F3SharedMemoryListeningConnector - currently only available on Windows

    • The connector object can be used to obtain an {@link org.f3.jdi.F3VirtualMachine} as described in the JDI documentation.

    This javadoc shows the classes and methods that F3-JDI adds to JDI, as well as the enhancements to existing JDI methods. Due to limitations of javadoc, the existing unmodified methods in these classes are also shown. These methods have no javadoc comments, and behave as described in the JDI spec.
